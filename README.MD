# QA API Automation - CRUD Categories (Learning Project)

## 📌 About

This project was developed as a learning exercise to practice API automation using Supertest and Jest.

It focuses on CRUD operations, authentication flow, and basic test structure organization.

---

## 🎯 What I Practiced

- API testing with Supertest
- CRUD operations (GET, POST, PUT, DELETE)
- Authentication using JWT tokens
- Test organization with Jest
- Setup/teardown using `beforeAll` and `afterAll`

---

## 🔄 CI/CD (GitHub Actions)

This project includes integration with GitHub Actions, automatically running the test suite on every push or pull request to the master branch.

### ⚙️ What the Pipeline Does

- Installs project dependencies
- Runs automated tests (Jest + Supertest)
- Generates an HTML report with the execution results

---

## ▶️ How to Run the Project

### 📦 Install dependencies

```bash
npm install
```

### 🚀 Run the tests

```bash
npm test
```

---

## 📊 Test Report (HTML)

After each pipeline execution, an HTML report is generated containing:

- Test status
- Execution details
- Failures and errors
- Test suite summary

---

### 📍 How to View the Report

1. Push changes or open a Pull Request to the `master` branch
2. Go to the **Actions** tab on GitHub
3. Select the latest workflow run
4. Download the HTML report artifact
5. Open the `index.html` file in your browser
